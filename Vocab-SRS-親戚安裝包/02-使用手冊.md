## 1. 切換使用者

右上角 **👤 圖示** → 點頭像選使用者。每個使用者的進度、SRS 排程獨立。

每個使用者可以自選顏色（藍、綠、黃、粉、紅、橘），右上角的小頭像會用對應的顏色標示。

---

## 2. 選等級

右上角 **Level 下拉選單**，從 **Level 1（最簡單）到 Level 6**。

| Level | 字數 | 難度 |
|-------|------|------|
| Level 1 | 1,018 | 國小 / 基礎 |
| Level 2 | 1,030 | 國中 |
| Level 3 | 1,053 | 國中進階 |
| Level 4 | 1,047 | 高中 |
| Level 5 | 1,055 | 高中進階 |
| Level 6 | 1,052 | 大學 / 全民英檢中高級 |
| **合計** | **6,255** | |

字庫已內建在程式碼裡，不用自己準備。每張卡含：英文單字、中文翻譯、英文字典定義、發音音檔（點喇叭可聽）。

## 3. 練習卡片（SRS 三個按鈕）

中間是英文單字，**按一下卡片翻到中文翻譯**。下方三個按鈕：

| 按鈕 | 意思 | SRS 行為 |
|------|------|---------|
| ❌ **忘記** | 不會 | 隔天再考 |
| 😐 **普通** | 想了一下才會 | 隔幾天再考 |
| ✅ **熟悉** | 秒答 | 拉長間隔，1 週、2 週、1 個月... |

**SRS = Spaced Repetition System（間隔重複系統）**，依據你的反應自動排下次複習日期。
- 按「忘記」的字隔天會再出現
- 按「熟悉」的字會慢慢拉長到 1 週、2 週、1 個月後才再考
- 系統用的是經典的 SM-2 演算法，跟 Anki 同源

**建議用法**：每天打開 App，先看「📊 今日要複習 N 張」，把這 N 張練完當天就結束。剛開始 N 會比較多，練 1–2 週後會穩定在每天 30–80 張左右。

---

## 4. 查看練習紀錄

右上角 **📊 圖示** → 看每日練習數量、總熟悉字數。

可以看到：
- 今天練了幾張
- 連續練習天數
- 每個 Level 的進度（已熟悉 / 總卡片數）

## 5. 加 / 改使用者

右上角 **👤 → 設定**：

- ➕ **新增使用者**（要 PIN）
- ✏️ **改名**（要 PIN）
- 🎨 **改顏色**（要 PIN）
- 🗑 **刪除**（要 PIN，**會刪光該使用者所有紀錄，無法復原**）

PIN 是 4–8 位數字，**設定一次後全家庭共用**。在任何裝置上要管理使用者都要打 PIN。同一裝置上打過一次 PIN 之後，下次不會再問（避免每次新增使用者都要重打）。

---

## 6. 切換家庭（很少用到）

右上角 **👤 → 設定 → 切換家庭**。

如果你同時管理多個家庭（例如自己家 + 表妹家），可以在這裡切換。一般用戶不會用到。

切換到別的家庭後，**會要求重新輸入該家庭的 PIN** 才能管理那邊的使用者。

---

## 7. 跨裝置同步

只要不同裝置開的是**同一個 `#fam=xxxxxxxx` 網址**，資料自動同步：

- 爸爸手機加了使用者 → 媽媽手機自動看到
- 小孩在 iPad 練了 20 個字 → 在 iPhone 也看到一樣的紀錄
- 改了使用者顏色、名字 → 全家裝置 1 秒內更新

同步是雲端即時推送的（用 Firebase Firestore），延遲約 1 秒。網路斷掉也能離線練，等網路恢復會自動 push 上去。

> ⚠️ 如果 PWA（桌面圖示）打開後家庭代碼不一樣，代表那台裝置裝錯了。要從含 `#fam=` 的網址重新裝一次。

## 8. 疑難排解 Q & A

### Q: 網址打開全白 / 一直轉圈

- 確認安裝第 5 步的 `firebaseConfig` **整段都換對了**，沒有少貼某幾行。
- 確認 Firestore 已建立（安裝第 3.2 步）。
- 用桌機 Chrome 開網址 → 按 F12 → 看 Console 紅字，那是錯誤訊息。

### Q: 加使用者沒反應 / 點下去沒動

- 重新整理頁面（iPhone Safari 是下拉刷新；PWA 圖示要關掉重開）。
- 確認 Firestore Rules 已部署（安裝第 4 步）。

### Q: 兩支手機看到的使用者不一樣

- 確認**兩支都用 `#fam=xxxxxxxx` 同一個網址**。
- 在 PWA 圖示（沒網址列那個）裡看不到家庭代碼，可以從右上角 👤 → 設定 → 家庭區塊看代碼是不是一致。

### Q: 忘記 PIN

目前沒有忘記 PIN 重設功能。如果真的忘了，要用瀏覽器直接到：

1. Firebase Console → Firestore Database
2. 找到 `families/你的家庭代碼/state/main` 文件
3. 把 `pinHash` 欄位改成空字串 `""`
4. 回到 App 就可以重新設 PIN

### Q: 我想改程式 / 修 bug

- 在 GitHub 網頁直接改 `docs/index.html` → Commit → 等 1 分鐘 GitHub Pages 自動重新部署。
- 如果改了 HTML/CSS/JS 結構，記得把 `docs/sw.js` 裡 `CACHE_VERSION` 數字往上加（例如 1.2.2 → 1.2.3），不然瀏覽器會吃舊版快取。

### Q: Firebase 會收錢嗎？

家庭使用量遠遠在免費額度（Spark Plan）內：

- 每日 50,000 次讀取（你全家一輩子也用不完）
- 每日 20,000 次寫入
- 1 GB 儲存空間

要超過很難，可以放心。如果你怕，到 Firebase Console → 左下齒輪 → 用量與計費 → 確認還在 Spark Plan 就好。

### Q: 手機上音檔不會發聲

- 點英文單字旁邊的喇叭圖示。
- iOS 第一次點要先解除靜音模式（側邊那個小撥桿）。
- 音檔是從免費字典 API 抓的 mp3，網路太慢可能要等 1–2 秒。

### Q: 想把某個字從牌組移除（已經學會了不想再看到）

目前 App 沒有「踢出牌組」按鈕。你可以連按「✅ 熟悉」幾次，SRS 會把間隔拉到很長（幾個月後才再考），實際上等於不會再看到。
